# T22 — Конструктор пайплайнов на канвасе

- **Веха:** M4
- **Зависит от:** T03, T05
- **Спека:** SDD FR-2.5–2.7

## Скоуп

- Палитра типов плиток; drag&drop на канвас создаёт шаг с уникальным id.
- Протяжка ребра между плитками = `depends_on`; удаление/дублирование плиток и рёбер.
- Undo/redo (командный стек над моделью).
- Live-валидация (валидатор T03) на каждое изменение графа; ошибки бейджами;
  запуск прогона заблокирован до зелёной валидации.
- Вкладка YAML: второе представление той же модели, переключение без потери правок;
  сохранение регенерирует YAML (комментарии не сохраняются — зафиксировано в FR-2.7);
  diff перед сохранением.
- Редактирование конфига шага в инспекторе (формы по типу: expects, allowed_write,
  env_scope, budget, retry, fail_policy, routes).

## Вне скоупа

- Библиотека плиток и создание agent-плитки с нуля (T23).

## Приёмка

- [ ] GWT FR-2.5–2.6: agent+judge собраны, без промпта — бейдж и блок запуска; после заполнения — зелёно
- [ ] пайплайн, собранный из палитры, исполняется end-to-end на фикстурном рантайме (SDD §7.6)
- [ ] undo/redo восстанавливает граф и конфиги без рассинхрона с YAML-вкладкой
