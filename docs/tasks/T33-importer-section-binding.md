# T33 — Импортёр: привязка по §-секции и неоднозначный матчинг

- **Веха:** M5
- **Зависит от:** T24
- **Спека:** SD §8, SDD FR-9; ревью импортёра 2026-07-11 №2

## Скоуп

- Неоднозначный автоматч: если id шага входит в заголовки нескольких секций,
  плитка не биндится молча первой (`findFirst`), а помечается «неоднозначно»
  со списком секций-кандидатов — пользователь выбирает.
- Точность матчинга: вхождение id как отдельного токена (границы слова)
  предпочитается substring-вхождению — `fp-red` не должен матчиться на заголовок
  про `fp-red-fix`, когда есть точное совпадение.
- Ручная привязка по секции: если выбранный файл — markdown с заголовками
  (например, `subagent-prompts.md`), диалог предлагает выбрать §-секцию, и
  плитке достаётся только её body. Обычный файл — как сейчас, целиком
  (`ImportSession.bindManually`).

## Вне скоупа

- Парсинг произвольной прозы SKILL.md в семантику шагов (SD §8, осознанный отказ).

## Приёмка

- [ ] две секции с упоминанием id → плитка подсвечена, кандидаты показаны, выбор фиксируется в биндинге
- [ ] ручной выбор §-секции даёт промпт, равный body секции, а не всему файлу
- [ ] id, являющийся префиксом другого id, матчится по токену корректно (тест на пару вида `fp-red`/`fp-red-fix`)
