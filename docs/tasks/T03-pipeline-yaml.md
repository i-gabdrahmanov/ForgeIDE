# T03 — pipeline.yaml: парсер и валидатор

- **Веха:** M1
- **Зависит от:** T02
- **Спека:** SD §5; SDD FR-2.1–2.3, §5.1

## Скоуп

- Маппинг YAML ↔ доменная модель (Jackson dataformat-yaml), включая поля безопасности:
  `allowed_write`, `env_scope`, `budget`, `retry`, тип `outward` с `actions`.
- Механизм подстановок `${project.*}/${feature.*}/${params.*}` — парсинг ссылок при загрузке,
  рендер значений на этапе исполнения (интерфейс `VariableResolver` в core).
- Валидатор: ацикличность (топосорт), достижимость всех шагов, существование ссылок
  `target`/`routes`/`prompt`-файлов, судья перед каждым outward-шагом, уникальность id.
- Ошибки валидации с координатой (id шага + поле) — структура для подсветки на канвасе.
- Шаблон `forgelite` в ресурсах (первая версия, по FORGE.md/pipeline-technical.md).

## Вне скоупа

- Канвас (T05), генерация YAML из канваса (T22).
