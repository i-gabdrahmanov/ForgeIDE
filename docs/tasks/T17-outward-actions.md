# T17 — Outward-шаги движка (push/PR/Jira)

- **Веха:** M3
- **Зависит от:** T06, T12
- **Спека:** SDD SR-4, §5.1 (тип `outward`); Т-4

## Скоуп

- Тип шага `outward`: исполняется движком детерминированным кодом, всегда после гейта
  (валидатор T03 это уже требует).
- Экшены: `git_push` (git CLI от имени IDE с кредами из конфига, не из env фаз),
  `create_pr` (Bitbucket/GitHub REST; target-ветка по конфигу, stacked по depends_on),
  `jira_comment`/`jira_transition` (REST).
- Пред-условия экшена: чистый scope-diff последней агент-фазы, PASSED всех судей ветки;
  нарушение → отказ с причиной.
- Результаты (URL PR, id комментария) → аудит + панель артефактов шага.
- Ошибки внешних систем (сеть, 4xx/5xx) → `FAILED(script)`-класс с ретраем по политике,
  идемпотентность: повторный push/PR не дублирует (проверка существования ветки/PR).

## Вне скоупа

- MCP-интеграции внутри агент-фаз (остаются как есть).

## Приёмка

- [ ] e2e на тестовом репозитории: ветка+push+PR создаются движком после подтверждения гейта
- [ ] у агент-фаз прогона нет кредов в env (совместно с T16, автотест)
- [ ] повторный запуск outward-шага после сбоя сети не создаёт дубликат PR
- [ ] отклонённый гейт → outward-шаг не исполняется (SKIPPED/ветка отклонения)
