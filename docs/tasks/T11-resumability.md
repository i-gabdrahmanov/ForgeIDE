# T11 — Резюмируемость и ретраи

- **Веха:** M2
- **Зависит от:** T06, T07
- **Спека:** SDD FR-3.4, FR-11.2, FR-11.4, NFR-3

## Скоуп

- Recovery при старте IDE: загрузка прогонов из SoT, `RUNNING` → `FAILED(interrupted)`,
  событие recovery в аудит, предложение ретрая в UI.
- Политика ретраев per-step (`retry: {stream: 1, script: 0}`) с глобальными дефолтами;
  авторетрай только для безопасных классов (stream, script).
- Ручной ретрай любого FAILED-шага из UI (кроме заблокированных инцидентами до разбора).
- `PAUSED(engine-error)`: перехват исключений актора, стейт не повреждается, после
  рестарта — штатный recovery.
- Проверка расхождения снапшота определения с диском при ретрае (FR-3.5) → предупреждение с диффом.

## Вне скоупа

- Эскалация судей (T14).

## Приёмка

- [ ] GWT FR-3.4 автотестом: kill -9 IDE в середине фазы → после старта PASSED сохранены, RUNNING → FAILED(interrupted), ретрай доступен
- [ ] восстановление прогона ≤ 5 с (NFR-3)
- [ ] `FAILED(stream)` ретраится автоматически один раз, второй сбой — терминален до ручного действия
