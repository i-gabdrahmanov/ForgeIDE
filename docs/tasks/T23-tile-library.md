# T23 — Библиотека плиток, плитка с нуля

- **Веха:** M4
- **Зависит от:** T22
- **Спека:** SDD FR-2.8, FR-2.9

## Скоуп

- Создание agent-плитки «с нуля»: генерация prompt-файла из шаблона с контрактной частью
  (итоговый JSON §5.2, `step_id`, запрет самостоятельных outward-действий); контрактная
  часть подсвечена, правка/удаление — с предупреждением (совместно с T20).
- Библиотеки плиток: проектная (`<project>/.forgeide/library/`) и пользовательская
  (`~/.forgeide/library/`); сохранение плитки или выделенного подграфа в библиотеку.
- Вставка из библиотеки в пайплайн: копирование промпта/конфига, перегенерация id,
  перевязка путей (prompt, скрипты) на структуру целевого проекта.
- Метаданные плитки в библиотеке: owner/validity/scope (модель реестра, совместимая с T24).

## Вне скоупа

- Общий (командный) реестр плиток с синхронизацией — пост-MVP.

## Приёмка

- [ ] созданная с нуля agent-плитка валидна и исполняется на фикстурном рантайме
- [ ] подграф (agent+judge+gate) сохранён в пользовательскую библиотеку и вставлен в другой проект без ручной правки путей
- [ ] попытка удалить контрактный блок промпта → предупреждение
