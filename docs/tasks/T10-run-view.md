# T10 — Run view: live-лог, timeline

- **Веха:** M2
- **Зависит от:** T05, T06
- **Спека:** SD §7; SDD FR-7.6–7.8, NFR-1, NFR-2

## Скоуп

- Оверлей статусов на канвасе: цвет/бейдж по `RunSnapshot` (включая счётчик итераций,
  бейдж «N вопросов» для WAITING_INPUT — сам диалог в T13).
- Live-лог идущей фазы: tailer на виртуальном потоке, батчинг 100 мс, кольцевой буфер
  10k строк на шаг; follow-режим.
- Вкладки лога шага: parsed-события / raw stdout / stderr / meta.json; полнотекстовый поиск.
- Timeline прогона: хронология событий аудита, фильтры (шаг, тип, «только инциденты»),
  клик → payload + ссылки на raw-логи.
- Экспорт прогона архивом (audit + meta + raw + вердикты) — FR-7.8.
- Запуск/отмена прогона из UI; список прогонов фичи (FR-7.9, без сравнения).

## Вне скоупа

- Диалоги гейтов и вопросов (T12, T13), метрики-дашборд (пост-MVP).

## Приёмка

- [ ] статус на канвасе обновляется ≤ 200 мс от события движка (NFR-2)
- [ ] gradle-подобный болтливый вывод не подвешивает UI (кольцевой буфер работает)
- [ ] фильтр «только инциденты» показывает tamper/scope/drift события
- [ ] экспортированный архив содержит всё для воспроизведения фазы (meta с полным промптом)
