# T36 — Предупреждение о грязном дереве и границы scope-diff

- **Веха:** M5
- **Зависит от:** T16
- **Спека:** SDD SR-6/Т-13, SD §9; ревью импортёра 2026-07-11 №5

## Скоуп

- Старт прогона на грязном git-дереве → предупреждение в UI + событие в аудит:
  scope-diff в этом режиме ослаблен — файл, уже грязный до фазы, агент может
  дописывать без детекта (осознанный трейд-офф `GitScopeDiff`, но пользователь
  должен его видеть, а не узнавать из Javadoc).
- Раздел «границы контроля» в документации (README — координация с T30, и/или
  docs/sd): scope-diff не видит gitignored-пути (`git status` их не показывает),
  записи вне репозитория и сеть; сетевые/файловые ограничения внутри фазы —
  вторая линия хуков CLI (SD §9); превенция вместо детекта — T29 (PhaseSandbox).
  Рекомендация: запускать прогоны на чистом дереве.

## Вне скоупа

- Превенция записей (реальный sandbox — T29 и пост-MVP по SDD §8).
- Детект записей в gitignored-пути (потребовал бы полного обхода дерева, против NFR-4).

## Приёмка

- [ ] прогон на грязном дереве показывает предупреждение и пишет audit-событие (прогон не блокируется)
- [ ] прогон на чистом дереве — поведение и UX без изменений
- [ ] README/docs содержат раздел о границах scope-diff с перечнем слепых зон из ревью
