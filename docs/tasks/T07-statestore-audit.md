# T07 — StateStore: SoT + hash-chain аудит

- **Веха:** M2
- **Зависит от:** T02
- **Спека:** SD §4; SDD FR-7.1–7.3, SR-1, SR-3

## Скоуп

- Раскладка `~/.forgeide/state/<project-hash>/<pipeline>/<feature>/`: `run.json` + `audit.jsonl`.
- `run.json`: атомарная запись (tmp + rename) + контрольная сумма содержимого.
- `audit.jsonl`: конверт §5.3 (`seq, ts, run, step, iter, type, payload, prev, hash`),
  `hash = SHA-256(prev + canonical(payload))`; каноникализация JSON детерминированная.
- Загрузка прогона: верификация контрольной суммы run.json и целостности цепи аудита;
  разрыв → `STOPPED(audit-chain)`.
- Гарантия SR-1: пути SoT нигде не передаются в промпты/env агент-фаз (проверяется в T09/T16).
- Логи фаз: раскладка `ground/ai-logs/<feature>/iter-NN/<step>/` (запись — из T08/T09,
  здесь — структура каталогов и meta.json контракт §5.4).

## Вне скоупа

- Проекция манифеста для хуков (T15), recovery-логика (T11).

## Приёмка

- [ ] порча байта в середине audit.jsonl → STOPPED(audit-chain) при загрузке
- [ ] kill процесса в середине записи run.json → после рестарта валиден предыдущий снапшот (тест с прерыванием записи)
- [ ] последовательность из 10k событий пишется и верифицируется < 1 с
