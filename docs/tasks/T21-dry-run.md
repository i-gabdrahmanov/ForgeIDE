# T21 — Dry-run судьи + предпросмотр промпта

- **Веха:** M4
- **Зависит от:** T14, T20
- **Спека:** SDD FR-8.4, FR-8.5

## Скоуп

- Кнопка «прогнать судью» на judge-плитке: исполнение check-скрипта (и опционально LLM-судьи)
  на артефактах последней итерации целевого шага; вердикт показывается в панели,
  в SoT не сохраняется, на статусы шагов не влияет; в аудит — только событие `judge.dryrun`.
- Цикл «правка check-скрипта (T20) → dry-run → сохранить» без перезапуска агент-фаз.
- Предпросмотр рендера промпта: подстановки `${...}` + блоки `accumulated_errors`/`answers`
  ровно тем кодом рендера, что используется при боевом запуске (общий модуль — не копия).
- Dry-run доступен и при отсутствии прогона (по артефактам с диска, если пути существуют).

## Вне скоупа

- Dry-run целой агент-фазы (сознательно: реальный вызов модели = обычный запуск шага).

## Приёмка

- [ ] dry-run не меняет run.json и статусы; в аудите только judge.dryrun
- [ ] после правки скрипта dry-run немедленно отражает новое поведение (кэш T18 обновлён trusted-путём)
- [ ] предпросмотр промпта байт-в-байт совпадает с промптом реального следующего запуска (тест на общий рендер)
