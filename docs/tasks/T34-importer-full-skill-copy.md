# T34 — Импортёр: полное копирование каталога скилла

- **Веха:** M5
- **Зависит от:** T24
- **Спека:** SD §8, SDD FR-9/SR-8; ревью импортёра 2026-07-11 №3

## Скоуп

- `ImportSession.result()` переносит каталог скилла целиком
  (`SKILL.md`, `references/`, `scripts/`, вложенные файлы), а не только `SKILL.md`:
  сейчас скопированный скилл, ссылающийся на `references/…`, сломан для
  автодискавери в сессии gigacode — ссылки ведут на несуществующие файлы.
- Фильтрация мусора — тот же список, что у сканера (`ScaffoldScanner.SKIP_DIRS`:
  `.git`, `.venv`, `__pycache__`, …).
- Непривязанные к judge-плиткам `check_*.py` едут как часть каталога скилла.
- Предупреждение в UI при подозрительно большом скилле (> 10 МБ) — защита от
  случайного захвата бинарников/данных.

## Вне скоупа

- Выборочное копирование по ссылкам из SKILL.md (анализ прозы — SD §8).

## Приёмка

- [x] после импорта `.gigacode/skills/<id>/` пофайлово совпадает с источником (минус SKIP_DIRS)
- [x] SKILL.md со ссылкой на `references/…` разрешается в целевом проекте
- [x] хэш-манифест харнесса (SR-8) включает все скопированные файлы — drift на references детектится
