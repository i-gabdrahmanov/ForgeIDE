# T18 — Целостность обвязки: кэш, hash, preflight

- **Веха:** M3
- **Зависит от:** T07, T09
- **Спека:** SDD SR-7, SR-8, FR-1.4; Т-7

## Скоуп

- Деплой обвязки кнопкой: обёртка `deploy.sh <project>` + прогон `preflight.py`;
  вывод в UI, exit≠0 → запуск прогонов заблокирован (FR-1.4).
- IDE-кэш обвязки `~/.forgeide/harness-cache/<hash>/`: копия hooks+skills при деплое;
  скрипты судей и preflight исполняются из кэша, не из рабочей копии проекта (SR-7).
- Hash-манифест обвязки проекта (`hooks/`, `skills/`, `settings.hooks.json`): построение
  при деплое, сверка перед КАЖДОЙ агент-фазой; дрейф → `STOPPED(harness-drift)`,
  дифф изменённых файлов, действия «принять» (пересчёт эталона + аудит) / «откатить».
- API для trusted-правки из редактора (T20): атомарное обновление копия+кэш+hash
  с событием `harness.edited`.

- Сам редактор (T20).
