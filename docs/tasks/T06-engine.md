# T06 — PipelineEngine: актор и переходы

- **Веха:** M2
- **Зависит от:** T02
- **Спека:** SD §3; SDD FR-3

## Скоуп

- Актор: `BlockingQueue<EngineCommand>` + единственный поток мутаций; воркеры — виртуальные потоки.
- Жизненный цикл прогона: старт по фиче (slug+params), снапшот определения на старте (FR-3.5),
  вычисление готовности (`depends_on` в PASSED), диспатч исполнителям через порты.
- Обработка команд: `StepCompleted/StepFailed/GateAnswered/QuestionsAnswered/CancelRun/RetryStep`.
- Счётчик ре-итераций по FAIL судьи (лимит из FailPolicy), формирование `accumulated_errors`.
- Событийная шина: `RunSnapshot` после каждого перехода; персист через порт StateStore
  ДО публикации события (FR-3.3).
- Параллельные прогоны разных фич в одном движке.
- `PerTaskLoop`: разворачивание подграфа по task-plan.json после закрытия шага-источника.

## Вне скоупа

- Реализации портов (T07, T09), UI (T10), гейты/вопросы как диалоги (T12, T13).

## Приёмка

- [ ] property-style тесты переходов на фикстурных исполнителях (все статусы достижимы, недопустимые переходы невозможны)
- [ ] assert: мутации RunState только из потока-актора (проверка треда)
- [ ] два параллельных прогона не влияют друг на друга (тест)
- [ ] пайплайн из script-шагов проходит end-to-end на фикстурном ScriptRunnerPort
