# T35 — Импортёр: безопасный повторный импорт

- **Веха:** M5
- **Зависит от:** T20, T24
- **Спека:** SD §8, SDD FR-9; ревью импортёра 2026-07-11 №4

## Скоуп

- Перед записью `ImportWriter` сравнивает каждый целевой файл с новым содержимым
  и делит на new / identical / modified; identical не перезаписываются.
- Конфликты (modified — в т.ч. промпты, правленные через инспектор T20, и
  скрипты судей) показываются списком; без явного подтверждения пользователя
  файл не трогается. Опция «пропустить конфликтующие».
- Перезаписанные по подтверждению файлы сохраняются в бэкап
  `.forgeide/import-backup/<timestamp>/` (или в задаче фиксируется явное решение
  отказаться от бэкапа в пользу git).

## Вне скоупа

- Трёхстороннее слияние содержимого (merge промптов) — только detect + confirm.

## Приёмка

- [x] re-import без изменений в обвязке не перезаписывает ни одного файла
- [x] re-import при локально правленном промпте показывает конфликт; без подтверждения файл не изменён
- [x] после подтверждённой перезаписи прежняя версия доступна в бэкапе (либо решение об отказе зафиксировано в задаче)

## Реализация

- `ImportWriter.plan(projectRoot, result)` — сравнивает каждый целевой файл (`pipeline.yaml`,
  `import-manifest.json` и всё из `ImportResult#files()`, т.е. промпты/скрипты судей/скиллы/хуки/
  реестр) с уже лежащим на диске и возвращает `NEW` / `IDENTICAL` / `MODIFIED` (конфликт) — чистая
  функция, без записи.
- `ImportWriter.write(projectRoot, result, confirmedOverwrites)` — `NEW` пишется всегда,
  `IDENTICAL` не трогается вовсе (даже не перезаписывается тем же содержимым), `MODIFIED`
  перезаписывается только если его относительный путь в `confirmedOverwrites`; старое
  двухаргументное `write(projectRoot, result)` — это вызов с пустым множеством, т.е. режим
  «пропустить конфликтующие» по умолчанию.
- Перед перезаписью confirmed-конфликта прежнее содержимое файла сохраняется в
  `.forgeide/import-backup/<timestamp>/<относительный путь>` (один timestamp на весь `write()`).
- `ImportView.doImport()`: считает `plan()`, при отсутствии конфликтов импортирует как раньше; при
  наличии — показывает `ImportConflictDialog` (список конфликтов + «Перезаписать все» /
  «Пропустить конфликтующие» / «Отмена»), и только после явного выбора вызывает `write` с
  соответствующим множеством подтверждений.
- Тесты: `importer/.../ImportWriterTest.java` (plan/write/бэкап на уровне записи, без UI),
  существующий `ImportEndToEndTest` не менялся и остался зелёным.
