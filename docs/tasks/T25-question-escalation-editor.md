# T25 — Эскалация вопросов: открыть промпт в редакторе

- **Веха:** M5
- **Зависит от:** — (T13 и T20 смержены)
- **Спека:** SDD FR-10.5, FR-8.1–8.3; аудит 2026-07 №1 (`GateDialog.java:164`)

## Скоуп

- Заменить stub-кнопку `QuestionEscalationAction.OPEN_PROMPT` в `GateDialog` на реальный
  переход в редактор промпта (`TileEditorPanel`) целевой агент-фазы; `openPromptStubAlert`
  удалить.
- После сохранения промпта — путь к ретраю фазы прямо из контекста эскалации
  (кнопка «повторить с новым промптом» или бейдж на плитке).
- Сохранение идёт существующим trusted-путём FR-8.2 (`EngineCommand.PromptEdited` →
  аудит `prompt.edited`) — новых команд движка не заводить.
- Диалог эскалации при переходе в редактор закрывается штатно: шаг остаётся в своём
  статусе, эскалация переоткрывается с канваса (та же семантика, что отложенный ответ гейта).

## Вне скоупа

- Изменение лимита раундов вопросов (FR-10.5 остаётся ≤ 2).
- Действие «разбить шаг» — остаётся ручным (правка пайплайна в конструкторе).

## Приёмка

- [ ] кнопка «открыть промпт» из эскалации вопросов открывает редактор с промптом целевой фазы
- [ ] сохранение из этого пути даёт аудит-событие `prompt.edited` и действует со следующего запуска (тест)
- [ ] сценарий «эскалация → правка промпта → ретрай → фаза проходит» покрыт тестом на вынесенной логике
- [ ] grep по `stub` в main-коде `ui` пуст
