# T08 — ProcessRunner: stdio, process group, kill

- **Веха:** M2
- **Зависит от:** T01
- **Спека:** SD §6.1; SDD SR-9, NFR-1

## Скоуп

- Запуск подпроцесса в **новой process group** (POSIX setsid-семантика).
- stdin: запись промпта + закрытие потока; аргументы командной строки без текста промпта.
- Два pump-потока (виртуальные) на stdout/stderr, `redirectErrorStream(false)`;
  синхронный append raw-строк на диск (дренаж пайпа), затем разбор.
- Толерантный JSONL-парсинг: JSON → событие, не-JSON → `RawLine`, парсер не падает.
- Завершение: `waitFor()` + join обоих pump'ов; критерий успеха отдаётся вызывающему.
- Kill: по группе (`kill -PGID`) + `ProcessHandle.descendants()`; sweep осиротевших процессов
  с cwd в целевом проекте после фазы → kill + инцидент.
- Капы: wall-clock таймаут, лимит байтов вывода (дефолт 512 МБ) → kill + типизированная причина.
- Env: передаётся только явно переданный map (никакого наследования полного окружения IDE).

## Вне скоупа

- Знание о рантаймах/stream-json семантике конкретных CLI (T09).

## Приёмка

- [x] анти-дедлок тест: child пишет >64 КБ в оба потока одновременно — не виснет
- [x] child запускает `nohup`-внука — после kill фазы внук мёртв (или пойман sweep'ом)
- [x] превышение капа вывода/таймаута → kill группы + корректная причина
- [x] пропускная способность ≥ 10 МБ/с на синтетическом болтливом child (NFR-1)
